## Project Overview

`@qbtlabs/openmm-mcp` is an MCP (Model Context Protocol) server that exposes cryptocurrency trading, market data, and strategy tools to AI agents. It supports two deployment modes: **local** (stdio transport, user's own API keys) and **hosted** (HTTP transport on mcp.openmm.io with x402 USDC micropayments on Base).

## Architecture

### Entry Points

- **`src/index.ts`** ‚Äî Routes between setup wizard (`--setup`) and normal server startup. Exports `createServer()` for programmatic use.
- **`src/server.ts`** ‚Äî Core MCP server creation: registers tools/resources/prompts, handles stdio and HTTP transports, auto-loads vault credentials on startup. HTTP mode adds `/health` and `/mcp` endpoints with optional x402 payment gating.
- **`src/worker.ts`** ‚Äî Cloudflare Workers entrypoint for hosted deployment. Defers heavy imports inside request handlers to avoid Workers global-scope I/O restrictions.

### Tool System (`src/tools/`)

Each tool category exports a `register*Tools(server, exchangeManager)` function, aggregated in `src/tools/index.ts`:

- **market-data.ts** ‚Äî `get_ticker`, `get_orderbook`, `get_trades`, `get_ohlcv`
- **account.ts** ‚Äî `get_balance`, `list_orders`
- **trading.ts** ‚Äî `create_order`, `cancel_order`, `cancel_all_orders`
- **cardano.ts** ‚Äî `get_cardano_price`, `discover_pools` (Iris API for DEX pools)
- **strategy.ts** ‚Äî `start_grid_strategy`, `stop_strategy`, `get_strategy_status` (grid trading with linear/geometric spacing, flat/pyramidal sizing)

All tools use Zod schemas from `src/utils/validators.ts` for parameter validation. Tools return `{content: [{type: 'text', text: JSON.stringify(...)}]}`.

### Payment Layer (`src/payment/`)

`wrapServerWithPayment()` intercepts tool calls with x402 payment verification. Two modes:
- **Direct key mode** ‚Äî uses `WALLET_PRIVATE_KEY` env var
- **Signer mode** ‚Äî isolated x402-signer process (more secure, credentials never in MCP process)

Tool pricing: free (list_exchanges), read ($0.001), write ($0.01).

### Exchange Integration (`src/exchange/exchange-manager.ts`)

Wraps the `@3rd-eye-labs/openmm` SDK's `ExchangeFactory`. Supported exchanges: MEXC, Gate.io, Kraken, Bitget. `getConnectorSafe()` provides graceful error handling around SDK calls.

### Credential Management

- **Vault** (`src/vault/`) ‚Äî AES-256-GCM encrypted storage at `~/.openmm/vault.json`. CLI via `openmm-vault`.
- **Credentials Server** (`src/credentials/`) ‚Äî Isolated process serving credentials via Unix socket (`/tmp/x402-signer.sock`). CLI via `openmm-creds`.
- Auto-loaded at startup via `src/vault/loader.ts`, setting env vars for exchange adapters.

### Resources & Prompts

- `src/resources/` ‚Äî MCP resources listing supported exchanges and strategy templates
- `src/prompts/` ‚Äî Trading and market analysis prompts providing guardrails for AI agents

## Key Patterns

- **Symbol format**: Accepts `BTC/USDT` or `BTCUSDT`, normalized to uppercase
- **Cardano price**: Averages ADA/USDT from multiple CEX sources (Binance, MEXC, Coingecko)
- **Grid strategy**: Supports geometric spacing (cumulative factor) and pyramidal sizing (larger orders near center)
- **Smithery**: `createSandboxServer()` scans tools without real credentials for registry publication
- **Transport detection**: `MCP_TRANSPORT` env var selects stdio vs HTTP; x402 enabled when HTTP + payment env vars present

## TypeScript Configuration

- Target: ES2020, Module: NodeNext, strict mode
- Tests use ts-jest with ESM transforms for `@noble/*` and `@qbtlabs/x402` packages
- x402 modules have manual mocks in `src/tests/__mocks__/` to prevent real payment calls in tests

## Security Rules
- Vault password is interactive-only ‚Äî never in CLI flags, env vars, or config files
- No credentials, private keys, or socket paths in source files
- Socket default: `/tmp/openmm.sock` via `OPENMM_SOCKET` env var (replaces the two-socket model)

## What Claude Gets Wrong Here
- Do NOT suggest storing OPENMM_PASSWORD in claude.json or any config ‚Äî defeats the vault
- Do NOT use `require()` ‚Äî this project is ESM (NodeNext modules)
- Run `npm run typecheck` after any series of changes, not just at the end
